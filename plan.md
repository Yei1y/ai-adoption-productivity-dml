### 3. 数据与变量设计

#### 数据集
**Global AI Adoption & Workforce Impact Dataset**（Mohan Krishna Thalla）
- 约 150,000 条企业级观测
- 面板结构，跨 2023-2026 年
- 覆盖多个国家和行业
- 包含 AI 采纳、企业绩效、劳动力、治理等多维度变量

#### 变量定义

| 变量类型 | 变量名称 | 构造方式 | 选择理由 |
|---------|---------|---------|---------|
| **处理变量 D** | AI 采纳程度 | 将 AI adoption stage 二值化为高采纳 vs 低/无采纳 | 核心解释变量，反映企业 AI 化水平 |
| **结果变量 Y** | 劳动生产率 | Revenue / Employees | 经济学中最常用的生产率度量（Syverson, 2011） |
| **协变量 X** | 企业特征 | 行业、国家、企业规模、R&D 投入、自动化率、劳动力技能、治理指数、创新指数等 | 同时影响 AI 采纳和生产率的混淆因素，需用 ML 灵活控制以实现条件独立性假设 |

#### 可视化计划
1. AI 采纳率的行业分布（柱状图）→ 展示行业异质性
2. AI 采纳 vs 非采纳企业的生产率分布对比（箱线图/密度图）→ 展示原始差异
3. 不同企业规模下 AI 采纳率（堆叠柱状图）→ 展示规模作为混淆因素
4. 主要连续变量的相关性热图 → 展示变量间关系，凸显 DML 必要性

### 4. 模型设定

#### Partial Linear Model（部分线性模型）

```
Y = θ₀D + g₀(X) + ε,      E[ε | X, D] = 0
D = m₀(X) + ν,            E[ν | X] = 0
```

其中：
- Y：结果变量（劳动生产率的对数）
- D：处理变量（AI 采纳状态，0/1）
- X：高维协变量向量
- g₀(·) 和 m₀(·)：未知的 nuisance 函数，由 ML 估计
- θ₀：待估因果参数（ATE）

#### DML 估计量（Neyman 正交评分）

基于部分线性模型的 Neyman 正交评分函数：

```
ψ(W; θ, η) = (Y - ℓ₀(X) - θ(D - m₀(X))) · (D - m₀(X))
```

其中 η = (ℓ₀, m₀)，ℓ₀(X) = E[Y|X]，m₀(X) = E[D|X]

正交性保证：∂E[ψ(W; θ₀, η₀)] / ∂η = 0，即 nuisance 参数的估计误差对因果参数 θ 的影响是二阶小量。

#### 交叉拟合（Cross-fitting）流程

1. 将样本随机等分为 K 折（K = 5）
2. 对于每折 k = 1, ..., K：
   - 在其余 K-1 折上估计 ℓ̂₀^(-k)(X) 和 m̂₀^(-k)(X)
   - 在第 k 折上计算残差：Ỹ = Y - ℓ̂₀^(-k)(X)，D̃ = D - m̂₀^(-k)(X)
   - 用 OLS 估计：θ̂_k = (D̃'D̃)⁻¹D̃'Ỹ
3. 聚合估计：θ̂ = (1/K) Σ θ̂_k
4. 推断：渐近正态分布，采用稳健标准误

#### ML 方法选择

| 方法 | 用途 | 优势 |
|------|------|------|
| **LASSO** | Nuisance 函数估计 | 高维稀疏设定下表现好，计算快 |
| **Random Forest** | Nuisance 函数估计 | 捕捉非线性关系和交互效应 |
| **Gradient Boosting (XGBoost)** | Nuisance 函数估计 | 灵活，预测精度高 |
| **组合（SL）** | 稳健性检验 | 集成多种方法降低模型选择风险 |

### 5. 编码与分析分离的工作流

```

                              ↓

```

### 6. 稳健性检验方案

1. **更换 ML 方法**：分别用 LASSO、RF、XGBoost 估计 nuisance 函数，比较结果
2. **改变样本分割方式**：尝试 K = 2, 5, 10，检验对折数的敏感性
3. **替换处理变量定义**：改变二值化阈值，检验定义敏感性
4. **替换结果变量**：尝试其他生产率指标
